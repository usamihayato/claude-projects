## 7. 評価スコアリングシート

### 影響調査（カテゴリA・B）スコアリング

```
正解率 = 正解ケース数 / 全ケース数 × 100 (%)

再現率 = 正解セットのうち回答に含まれた件数の平均
       = Σ(正解件数に含まれた件数 / 正解全件数) / ケース数 × 100 (%)

適合率 = 回答のうち正解だった件数の平均
       = Σ(正解件数に含まれた件数 / 回答件数) / ケース数 × 100 (%)
```

### ソースコード解説（カテゴリC・D）スコアリング

```
平均関連性スコア = Σ(個別スコア) / ケース数  (1〜5点)
```

### ツール選択精度（カテゴリE）スコアリング

```
ツール選択精度 = 期待どおりのツールを使用したケース数 / 全ケース数 × 100 (%)
```

## 9. 評価実施手順

### Step 1: テスト環境準備

```sql
-- テストケース記録テーブルを作成
CREATE OR REPLACE TABLE T_EVAL_RESULTS (
    eval_id       NUMBER AUTOINCREMENT,
    test_case_id  VARCHAR(10),   -- 例: A-01
    category      VARCHAR(5),
    question      VARCHAR(2000),
    tool_type     VARCHAR(50),   -- search_only / hybrid
    used_tool     VARCHAR(100),  -- 実際に使われたツール
    answer        VARCHAR(10000),
    expected      VARCHAR(10000),
    is_correct    BOOLEAN,
    recall_score  FLOAT,
    precision_score FLOAT,
    relevance_score INTEGER,     -- 1〜5 (C/D/E)
    tool_match    BOOLEAN,       -- ツール選択が期待通りか
    response_sec  FLOAT,
    eval_date     TIMESTAMP DEFAULT CURRENT_TIMESTAMP()
);
```

### Step 2: 自動実行スクリプト

```python
# テストケースを1件ずつ実行し結果をT_EVAL_RESULTSに記録
for case in test_cases:
    start = time.time()
    response = call_agent(case["question"])
    elapsed = time.time() - start
    
    # 結果をパース・記録
    insert_eval_result(
        test_case_id=case["id"],
        answer=response["text"],
        used_tool=response["tool_used"],
        response_sec=elapsed
    )
```

### Step 3: 人手評価

- カテゴリC・D・Eの関連性スコアは評価者2名が独立評価
- スコア差が2点以上の場合は協議して確定
- 影響調査（A・B）は自動判定（期待回答との完全一致チェック）

### Step 4: 集計・比較

```sql
-- カテゴリ別スコアを集計
SELECT
    category,
    tool_type,
    COUNT(*) AS total_cases,
    AVG(CASE WHEN is_correct THEN 1 ELSE 0 END) * 100 AS accuracy_pct,
    AVG(recall_score) * 100 AS avg_recall_pct,
    AVG(relevance_score) AS avg_relevance,
    AVG(response_sec) AS avg_response_sec
FROM T_EVAL_RESULTS
GROUP BY category, tool_type
ORDER BY category, tool_type;
```
