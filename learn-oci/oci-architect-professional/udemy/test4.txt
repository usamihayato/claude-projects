A: 重要： Data Guard は Oracle Database 専用の技術 であり、MySQL Database Service では使用できません。このため、A は技術的に不可能です。

B: 「手動スケーリング（Manual scaling）」は小売業の突発的なピークには不向きであり、設問の「シームレスなスケーラビリティ」という要件を満たせません。

C: 有効なアプローチですが、設問が重視している「災害復旧（DR）」への言及が D に比べて不足しています。

【重要ポイント】

「MySQL Database Service の DR」には Data Guard ではなく、リード・レプリカや受信レプリケーション、あるいはクロスリージョン・バックアップ・コピーを使用する。
