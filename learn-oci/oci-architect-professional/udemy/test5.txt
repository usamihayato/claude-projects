問題1
不正解
レガシーアプリケーションを Oracle Cloud Infrastructure (OCI) に移行して、そのパフォーマンスと信頼性を向上させる文脈において、ビジネス要件と技術要件の両方を満たすために最も効果的な 2 つのソリューションはどれですか？アプリケーションにはステートレス（状態を持たない）コンポーネントとステートフル（状態を持つ）コンポーネントが混在しており、高可用性が必要で、需要に基づいてスケーリングする必要があると想定します。

選択は正解です
E. データ管理には OCI Autonomous Transaction Processing (ATP) を、アプリケーションのデプロイには OCI Container Engine for Kubernetes を採用する。

B. 管理を簡素化するために、アプリケーション全体を単一の OCI Compute インスタンスにデプロイする。

正しい選択
C. ステートレスなコンポーネントのデプロイには OCI Container Engine for Kubernetes (OKE) を、ステートフルなコンポーネントのデータには OCI File Storage を利用する。

選択は不正解です
A. ステートレスなコンポーネントには OCI Functions を、ステートフルなコンポーネントのデータ永続化には OCI Block Volume を使用してアプリケーションを実装する。

D. 最大限のパフォーマンスと隔離性を確保するために、両方のコンポーネントに OCI Dedicated Compute Hosts を介した専用物理サーバーを採用する。



全体的な説明
【詳細解説】

正解（C, E）の根拠: * OKE (C, E): コンテナ化（OKE）は、ステートレスなアプリ層において迅速な水平スケーリングと高可用性を実現するデファクトスタンダードです。

File Storage (C): OCI File Storage (FSS) は、複数のコンピュート・インスタンスから同時にアクセス可能な共有ファイルシステムを提供します。これは、ログや共有構成を必要とするステートフルなアプリコンポーネントの可用性を高めるのに最適です。

ATP (E): データベースが必要なステートフルなワークロードに対し、Autonomous Transaction Processing (ATP) は自律的なスケーリング、自動バックアップ、高可用性を提供するため、管理負荷を抑えつつレガシーからの近代化に最も適しています。

不正解の理由:

A: OCI Block Volume は通常、一度に 1 つのインスタンスにのみアタッチされます（読み書き時）。複数のスケーリングされたインスタンス間でステートフルなデータを「共有」するには、FSS の方が適しています。

B: 「単一インスタンス」は、単一障害点（SPOF）となり、高可用性とスケーラビリティの要件を全く満たしません。

D: 専用ホストは物理的な隔離には有効ですが、それ自体が自動スケーリングや高可用性を担保するわけではなく、コスト効率も悪くなります。

【重要ポイント】

「コンテナ (OKE) ＋ 共有ファイル (FSS) ＋ 自律型 DB (ATP)」の組み合わせは、OCI におけるモダンな高可用性アプリ構成の鉄板。

問題4
不正解
ある金融機関が、データ分析および処理ニーズのために OCI の活用を検討しています。その機関は、大量のデータを効率的に処理し、リアルタイム分析を提供し、予測分析のための機械学習機能をサポートするソリューションを必要としています。どの OCI サービスまたは機能が彼らの要件に最も適していますか？

B. OCI Data Science は、データサイエンティストが OCI 上で機械学習モデルを構築、トレーニング、管理することを可能にし、予測分析能力を強化するが、リアルタイム分析には焦点を当てていない。

正解
A. OCI Autonomous Data Warehouse (ADW) は、分析に最適化された完全に管理されたスケーラブルなデータベースを提供し、組み込みの機械学習と高性能クエリをサポートしている。

回答は不正解です
D. OCI Big Data Service は包括的なビッグデータ・プラットフォームを提供するが、リアルタイム分析や予測機械学習よりも、大規模なデータセットの処理と分析に重点を置いている。

C. OCI Object Storage は、あらゆるタイプのデータに対して高度にスケーラブルなストレージを提供するが、主に包括的な分析や機械学習プラットフォームではなくストレージソリューションとして機能する。

全体的な説明
【詳細解説】

正解（A）の根拠: ADW は、データウェアハウス（分析）に特化した自律型データベースです。インメモリ処理による高速な分析クエリに加え、Oracle Machine Learning (OML) がデータベースに組み込まれており、SQL を使用してデータ移動なしにモデルの構築・予測が可能です。これにより、分析、リアルタイム性、機械学習の 3 要素を 1 つのプラットフォームで満たせます。

不正解の理由:

B: Data Science サービスはモデル構築には優れていますが、データの「管理（DB）」や「リアルタイムなクエリ応答」を提供する基盤ではありません。

C: Object Storage はデータレイクの保存先にはなりますが、それ自体に処理能力はありません。

D: Big Data Service は Hadoop/Spark 環境であり、SQL ベースの即時分析や組み込み ML の利便性では ADW に劣ります。

【重要ポイント】

「分析 ＋ 機械学習 ＋ 運用自動化」が必要なワークロードには Autonomous Data Warehouse (ADW) を選ぶ。

問題7
不正解
Oracle Cloud Infrastructure (OCI) デプロイメントのセキュリティ体制を強化するための推奨される手法に該当しないものはどれですか？

C. 仮想クラウドネットワーク (VCN) に対してきめ細かなイングレス（受信）およびエグレス（送信）ルールを定義するために、ネットワーク・セキュリティ・グループ (NSG) やセキュリティ・リストを実装する。

回答は不正解です

正解
D. データが機密性の高いものである場合でも、高速なパフォーマンスを確保するために、データ転送に暗号化されていない接続を定期的に使用する。

不正解の理由:

A: IAM と最小権限の原則は、セキュリティの根幹です。

B: WAF は、SQL インジェクションや XSS などの L7 攻撃から Web アプリを保護する標準的な手法です。

C: NSG やセキュリティ・リストは、VCN のマイクロセグメンテーションを実現するために不可欠な防御層です。

【重要ポイント】

OCI のセキュリティ設計において「暗号化」は譲れない基準であり、パフォーマンスを理由に省略してはならない。

問題12
不正解
ある国際的な小売企業が、ITインフラを統合し、OCIのグローバルなスケーラビリティと高度な分析機能を活用するために、世界中のさまざまなデータセンターから分散データベースと顧客データプラットフォームを Oracle Cloud Infrastructure (OCI) に移行することを計画しています。移行戦略では、データ損失ゼロを確保し、グローバルなデータプライバシー規制への準拠を維持し、顧客向けサービスへの影響を最小限に抑える必要があります。OCIへのデータベースおよびデータの移行を正常に実装し、トラブルシューティングするために不可欠な2つのアクションはどれですか？

正しい選択
B. 異種混合データベース間でのリアルタイムのデータ統合とレプリケーションに Oracle GoldenGate を利用し、OCIへの移行中も継続的な可用性と最小限のダウンタイムを確保する。

C. 移行プロセスを促進するためにパブリックインターネット接続を介して一括データ転送を実装し、OCIへの迅速な統合を達成するためにセキュリティよりも速度を優先する。

選択は正解です
D. OCIの Database Migration Service (DMS) を活用してリレーショナルデータベースの移行プロセスを自動化し、一貫性のある安全で効率的な転送を確実にすると同時に、最終的な切り替えまでリアルタイムのデータレプリケーションを有効にする。

E. 移行中は FastConnect のようなOCI専用の接続ソリューションの実装を避け、シンプルさとコスト削減のためにインターネット経由の VPN のみに依存する。

A. 移行プロセスを完全に制御するために手動のデータベース移行方法を選択し、各データベース転送を手動でスクリプト化し、データの整合性検証をデータベースダンプに依存する。

全体的な説明
【詳細解説】

正解（B, D）の根拠: *

B: Oracle GoldenGate は、異なるベンダー間（異種混合）やバージョン間でのリアルタイムデータ同期のデファクトスタンダードです。「データ損失ゼロ」と「最小限のダウンタイム」を求める大規模な分散環境において、DMS のバックエンド技術としても、あるいはスタンドアロンとしても不可欠なコンポーネントです。

D: OCI Database Migration Service (DMS) は、Oracle Zero Downtime Migration (ZDM) エンジンをベースにしたマネージドサービスです。オンライン移行（リアルタイムレプリケーション）をサポートしており、移行中のアプリケーション停止時間を最小化（またはゼロに）しつつ、整合性を保った自動移行が可能です。

不正解の理由:

A: 手動スクリプトとダンプによる移行はヒューマンエラーのリスクが高く、大規模な環境では非効率であり、ダウンタイムも長くなるため「最小限の影響」という要件に反します。

C: セキュリティよりも速度を優先し、パブリックインターネットで生データを送ることは、小売業が遵守すべきデータプライバシー規制（GDPR等）に真っ向から違反するアンチパターンです。

E: 大規模なグローバル移行において、不安定なインターネット VPN のみに頼ることは信頼性の欠如を招きます。FastConnect のような専用線は、移行の安定性と速度を担保するために強く推奨されます。

【重要ポイント】

「ダウンタイム最小化・データ損失ゼロ」の移行キーワードは、Database Migration Service (DMS) と GoldenGate である。

問題13
不正解
OCI ベースの eコマースプラットフォームの高可用性アーキテクチャを計画する際、複数の Web サーバーにトラフィックを分散させるために Oracle Cloud Infrastructure (OCI) Load Balancer を実装することにしました。システムのレジリエンス（回復力）をさらに高めるために、これらのサーバーを同じリージョン内の複数の可用性ドメインにデプロイしました。しかし、実装後、1 つの可用性ドメインで停止が発生した際に、プラットフォーム全体がオフラインになったことがわかりました。この問題の最も可能性の高い理由は何ですか？

A. ロードバランサのセッション維持（セッション・パーシステンス）構成が正しく設定されていなかったため、停止中にセッション情報が利用できなくなった。

B. 残りの可用性ドメインの Web サーバーが、1 つの可用性ドメインの停止によって増加した負荷を処理できるように適切にスケーリングされていなかった。

正解
C. ロードバランサのバックエンドセットの誤設定により、すべてのトラフィックが停止が発生した可用性ドメインのサーバーに向けられていた。

回答は不正解です
D. ロードバランサがヘルスチェックにおいて「fail-closed（異常時にすべて遮断）」設定で構成されていたため、いずれかの可用性ドメインで停止を検出すると、自動的にすべてのサーバーへのトラフィックを停止した。

全体的な説明
【詳細解説】

正解（C）の根拠: ロードバランサが論理的に複数 AD に跨っていても、バックエンドセットに登録されているサーバーが特定の AD のものだけに偏っていた、あるいは重み付け設定ですべてのトラフィックがその AD に行くようになっていた場合、その AD が落ちると代わりの転送先がなくなり、プラットフォーム全体がダウンします。

不正解の理由:

A: セッション情報の喪失は、ユーザーに再ログインを強いるなどの影響を与えますが、Web サイト自体がオフラインになる直接的な原因ではありません。

B: スケーリング不足（キャパシティ不足）の場合、サイトは「非常に重くなる」か、一部のリクエストがエラーになりますが、正常なサーバーが他 AD に存在するなら「完全なオフライン」とは通常表現されません。

D: OCI ロードバランサに「1 つが異常なら全部止める」という fail-closed 設定は存在しません。標準的な動作は、異常なインスタンスを切り離し、正常なものへ転送（Fail-open）することです。

【重要ポイント】

複数 AD 構成にしても、Load Balancer の「バックエンドセット」に全 AD のサーバーが正しく含まれていなければ意味がない。

問題21
不正解
ある金融機関が、Oracle Cloud Infrastructure (OCI) にビッグデータ・プラットフォームをデプロイして、データ分析機能の強化を計画しています。このプラットフォームは、リアルタイムの分析とレポート作成のために膨大な量の金融取引と顧客データを処理します。同社は、金融規制を遵守しつつ、最適なパフォーマンス、スケーラビリティ、およびデータセキュリティを達成することを重視しています。金融分析のために OCI にビッグデータ・プラットフォームをデプロイすることに関する記述として、「正しくない」ものはどれですか？

D. OCI の Big Data Service を活用すると、完全に管理された Hadoop および Spark 環境が提供され、ワークロードの需要に合わせて自動的にスケーリングでき、分析タスクの高いパフォーマンスが保証される。

正解
C. データの所在地（ローカリティ）を確保し、レイテンシを短縮し、グローバルな金融データ主権法に準拠するために、複数のリージョンと可用性ドメインにわたってビッグデータ・プラットフォームをデプロイすることが推奨される。

B. OCI Vault をビッグデータ・プラットフォームと統合することで、暗号化キーとシークレットが安全に管理され、データセキュリティと金融規制への準拠が強化される。

回答は不正解です
A. OCI Data Flow を使用すると、クラスター管理を必要とせずに Apache Spark アプリケーションを実行でき、データ処理ワークロードの運用効率とスケーラビリティを最適化できる。

全体的な説明
【詳細解説】

正解（C）が「正しくない」理由: 文脈に矛盾があります。

データ主権法: 多くの国の金融規制（データ主権法）では、データはその国（特定のリージョン）の外に出てはならないと定められています。複数のリージョンに分散させることは、かえって規制違反になるリスクがあります。

データ所在地: 「データの所在地を確保する」のであれば、特定の場所に固定すべきであり、複数リージョンへの分散とは相反する概念です。 高可用性のために複数 AD/リージョンにデプロイすることは技術的に可能ですが、主権法への準拠という文脈で「推奨される」という記述は不適切です。

不正解の理由（正しい記述）:

A: OCI Data Flow はサーバーレス Spark サービスであり、正しい説明です。

B: OCI Vault によるキー管理は金融業界のセキュリティ要件において標準的なプラクティスです。

D: Big Data Service (BDS) はマネージド Hadoop/Spark サービスであり、正しい説明です。

【重要ポイント】

「データ主権（Data Sovereignty）」が求められる場合、データは特定のリージョン内に留める必要がある。

問題23
不正解
国際的な小売企業が、eコマースプラットフォームを新しいグローバル市場に拡大しようとしています。プラットフォームは、変動する大規模なカスタマー・トラフィックを処理し、リージョンごとのデータ所在地（レジデンシ）コンプライアンスを確保し、高可用性と災害復旧能力を維持する必要があります。同社は、この拡張のために OCI を活用することを決定しました。アーキテクトが設計に含めるべき、ビジネスおよび技術要件を満たすための 2 つのアーキテクチャ上の決定はどれですか？

D. 地理的なユーザーの場所に基づいて、着信トラフィックを複数のリージョンに分散させるために OCI のグローバル・ロードバランサを実装する。

C. 管理とスケーラビリティを簡素化するために、すべてのリージョンにわたって統一されたデータベースとして OCI Autonomous Transaction Processing (ATP) を使用する。

E. 各リージョンの需要に基づいてリソースを自動的にスケーリングするために、OCI Identity and Access Management (IAM) ポリシーを構成する。

A. パフォーマンスと信頼性を確保するための直接的で高速な接続のために、OCI FastConnect を備えた単一リージョン・アーキテクチャを確立する。

【詳細解説】

正解（B, D）の根拠: *

B: リージョンごとに Object Storage を配置し、データをローカルに保持することは**「データ所在地（レジデンシ）」**への準拠に不可欠です。同時に、クロスリージョン・レプリケーションを組むことで DR 能力を確保します。

D: OCI Traffic Management（グローバル・ロードバランサ） は、DNS レベルでユーザーの地理的場所に基づいたルーティング（ステアリング）を行い、低レイテンシとリージョン間の負荷分散を実現します。

不正解の理由:

A: 「単一リージョン」では、グローバルな低レイテンシ提供やリージョン障害への耐性という要件を満たせません。

C: ATP は単一リージョン内で動作するサービスです。「全リージョンに跨る 1 つの ATP インスタンス」という構成は現時点で存在せず（Data Guard 等で複製は可能）、レジデンシ要件にも反する可能性があります。

E: IAM ポリシーは権限管理用であり、リソースの「オートスケーリング」を直接実行する機能ではありません（オートスケーリングはインスタンス・プールのポリシー等で行います）。

【重要ポイント】

「グローバル展開 ＋ コンプライアンス」の解は、地域ごとのデータ保存と DNS ベースのグローバル負荷分散の組み合わせ。

問題48
不正解
あなたは金融サービス企業向けの災害復旧（DR）戦略を設計している OCI アーキテクトです。高可用性と耐障害性を確保するため、重要なアプリケーションをリージョン内の複数の可用性ドメイン（AD）にデプロイする計画です。重要なデータのバックアップを保存するために OCI Object Storage を使用することに決めました。災害復旧目的での OCI Object Storage の使用に関する記述として、「正しくない（有効ではない）」ものはどれですか？

正解
A. OCI Object Storage は、地理的に離れたリージョンにデータの災害復旧用コピーを自動的に作成・管理する、組み込みの災害復旧ソリューションを提供している。

C. OCI Object Storage に保存されたデータは、高可用性を確保するために、リージョン内の複数の AD にある複数のストレージサーバー間で自動的に複製される。

回答は不正解です
B. 事前認証済みリクエスト (PAR) を使用すると、外部の当事者が OCI アカウントを持っていなくても、バケットやオブジェクトへのアクセスを安全に共有できる。

D. 誤った上書きや削除から保護するために OCI Object Storage でオブジェクト・バージョニングを有効にでき、災害復旧のための追加の保護手段を提供できる。

全体的な説明
【詳細解説】

正解（A）が正しくない理由: Object Storage は、同一リージョン内では自動的にデータを多重化（C）しますが、「別のリージョン」への自動的な DR コピーの作成は「デフォルト」ではありません。 リージョン跨ぎの複製が必要な場合は、ユーザーが手動で**「クロスリージョン・レプリケーション (Cross-Region Replication)」**を構成する必要があります。「自動的に（自動で）作成・管理する」という記述が誤りです。

不正解の理由（正しい記述）:

B: PAR は、有効期限付きの URL を発行して外部とデータを共有する標準機能です。

C: これが Object Storage の耐久性の根幹です。リージョン内で物理的に分散して保存されます。

D: バージョニングは、データの誤操作（論理的な災害）からの復旧に非常に有効です。

【重要ポイント】

Object Storage のリージョン内冗長性は自動だが、リージョン間冗長性（DR）は「レプリケーション・ポリシー」の手動設定が必要。
