# 01. Global Infrastructure

## 試験での重要度
- リージョン・AD・FDの概念と違いは必出
- 高可用性設計の問題で選択肢を絞る基礎知識

---

## 1. リージョン (Region)

- 世界中に分散した地理的な拠点（例: ap-tokyo-1, us-ashburn-1）
- 各リージョンは独立したインフラで構成される
- データは明示的に移動しない限り、リージョンをまたがない（データレジデンシー）
- 2025年時点で40以上のリージョンが存在

### リージョン選択の考慮点
| 観点 | 内容 |
|---|---|
| レイテンシ | ユーザーに近いリージョンを選択 |
| データレジデンシー | 法規制でデータ保管国が指定される場合 |
| サービス可用性 | 一部サービスは特定リージョンのみ提供 |
| 価格 | リージョンによって料金が異なる |

---

## 2. Availability Domain (AD)

- リージョン内に存在する**物理的に独立したデータセンター**
- 同一リージョン内のAD間は低レイテンシの専用ネットワークで接続
- 電源・冷却・ネットワークが完全に分離されており、1つのAD障害が他に影響しない
- リージョンにより1〜3個のADが存在
  - 東京リージョン（ap-tokyo-1）: AD × 1
  - Ashburn（us-ashburn-1）: AD × 3

### ポイント
- **高可用性を確保するにはリソースを複数ADに分散させる**
- ADが1つしかないリージョンでは、Fault Domainで可用性を確保する

---

## 3. Fault Domain (FD)

- 各AD内に存在する**論理的な障害分離グループ**
- 各ADに**3つのFD**が存在（FD-1, FD-2, FD-3）
- 同一FD内のサーバーはラック・電源・スイッチを共有する可能性がある
- 異なるFDに分散させることで、ハードウェア障害やメンテナンスの影響を限定できる

### 用途
- 1AD構成のリージョンで仮想マシンを3台配置する場合、それぞれ異なるFDに配置することで単一障害点を排除
- 計画メンテナンス時にOracleは同一FD内のインスタンスを同時には再起動しない

---

## 4. 階層構造のまとめ

```
Tenancy（テナンシー）
└── Region（リージョン）
    └── Availability Domain（AD）: 1〜3個
        └── Fault Domain（FD）: 3個固定
```

### AzureとOCIの対応表
| 概念 | Azure | OCI |
|---|---|---|
| 地理拠点 | Region | Region |
| 物理データセンター分離 | Availability Zone | Availability Domain |
| ラック・電源レベルの分離 | （Availability Zone内の内部概念） | Fault Domain |

---

## 5. 高可用性設計パターン

### マルチAD構成（推奨：3AD Regionの場合）
```
Region
├── AD-1: Webサーバー × 1, DBプライマリ
├── AD-2: Webサーバー × 1, DBスタンバイ
└── AD-3: Webサーバー × 1
```
- ロードバランサーをADをまたいで配置
- DBはData Guardでマルチレプリケーション

### シングルAD構成（1AD Regionの場合）
```
AD-1
├── FD-1: Webサーバー × 1
├── FD-2: Webサーバー × 1
└── FD-3: Webサーバー × 1
```

---

## 6. OCI固有の概念

### Realm（レルム）
- OCIの最上位の分離境界
- 商用クラウド（OC1）、政府クラウドなどが別Realmとして存在
- Realm内のリソースは互いに通信可能（適切な設定があれば）

### Home Region（ホームリージョン）
- テナンシー作成時に指定するリージョン
- IAM（Identity and Access Management）リソースはホームリージョンで管理される
- ホームリージョンは後から変更不可

## 8. よく出る問題パターン

**Q. 東京リージョン（1AD）で3つのVMを最も高可用に配置するには？**
→ A. 各VMを異なるFault Domain（FD-1, FD-2, FD-3）に配置する

**Q. IAMポリシーはどのリージョンに作成するか？**
→ A. ホームリージョンに作成する（他リージョンに自動レプリケーションされる）

**Q. ADが3つあるリージョンで最も高可用な構成は？**
→ A. リソースを3つのADにそれぞれ分散配置し、ロードバランサーで負荷分散する
